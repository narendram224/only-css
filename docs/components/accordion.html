Accordion

Accordions are collapsible content sections that allow users to expand and collapse information panels. Only-CSS provides a semantic accordion component using HTML5 details/summary elements with smooth animations.

Good to know: Accordions use semantic HTML5 <details> and <summary> elements, ensuring maximum accessibility and keyboard navigation without JavaScript.

Accessibility

Accessibility Features:
  • Semantic HTML5 details/summary elements
  • Keyboard navigation (Space, Enter, Tab)
  • Screen reader friendly
  • Focus states clearly visible
  • ARIA attributes automatically handled by browser

Best Practices

  • Use clear, descriptive summary text
  • Keep content sections focused and relevant
  • Consider the default state carefully
  • Test with screen readers and keyboard navigation
  • Use appropriate heading levels within content
  • Ensure sufficient color contrast for text

Design Guidelines

Pro Tip: Accordions work best for organizing large amounts of content or when you want to let users control what information they see. Don't use them for critical information that should always be visible.

When to Use Accordions

  • FAQ Sections: Organize frequently asked questions
  • Product Features: Group related features and details
  • Technical Documentation: Structure complex information
  • Settings Panels: Group configuration options
  • Content Organization: Break down long content into sections

When Not to Use Accordions

  • For critical information that must always be visible
  • When users need to compare information across sections
  • For very short content that doesn't need hiding
  • When the content hierarchy is unclear
  • For mobile devices where expanding might be cumbersome

Content Guidelines

  • Write clear, concise summary text
  • Keep related content together in sections
  • Use proper heading hierarchy within content
  • Consider the reading order and flow
  • Test with different content lengths
