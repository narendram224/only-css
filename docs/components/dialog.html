Dialog

Dialogs are modal windows that appear on top of the main content to capture user attention or prompt for input. Only-CSS provides a dialog component with backdrop, proper focus management, and accessibility features.

Good to know: Dialogs use semantic HTML5 dialog elements with proper ARIA attributes for maximum accessibility and screen reader support.

Accessibility

Accessibility Features:
  • Semantic HTML5 dialog element
  • Proper focus management
  • Escape key support
  • Screen reader friendly
  • Backdrop prevents interaction with background

Best Practices

  • Always provide clear dialog titles
  • Include close buttons or actions
  • Test with keyboard navigation
  • Ensure focus is properly managed
  • Use appropriate dialog types

Design Guidelines

Pro Tip: Use dialogs sparingly as they interrupt user flow. Consider using inline alternatives when possible.

When to Use Dialogs

  • Confirmations: Critical actions requiring confirmation
  • Forms: Complex data entry
  • Alerts: Important messages
  • Details: Additional information
