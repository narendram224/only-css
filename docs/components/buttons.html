Buttons

Buttons are the primary interactive elements in any interface. Only-CSS provides a comprehensive button system with multiple variants, sizes, and styles.

Good to know: Buttons use semantic HTML <button> elements by default, ensuring maximum accessibility and keyboard navigation.

Basic Usage

The simplest way to create a button is to use the <button> element:

Basic Button
<button class="oc-btn">Default Button</button>

Variants

Buttons come in several semantic variants using the data-variant attribute:

Button Variants
HTML 

<button class="oc-btn">Primary (Default)</button>
<button class="oc-btn" data-variant="secondary">Secondary</button>
<button class="oc-btn" data-variant="danger">Danger</button>
<button class="oc-btn" data-variant="link">Link</button>

Sizes

Buttons come in four predefined sizes:

Button Sizes
<button class="small">Small</button> <button class="oc-btn">Default</button> <button class="large">Large</button> <button class="xlarge">Extra Large</button>

States

Buttons automatically handle different states, including disabled state:

Button States
<button class="oc-btn" disabled>Disabled</button> <button class="oc-btn" data-variant="danger" disabled>Disabled Danger</button> <button class="outline" disabled>Disabled Outline</button>

API Reference

Attributes

Attribute Type Default Description
data-variant string "primary" Button variant: "primary", "secondary", "danger", "link"
disabled boolean false Disables the button and prevents interaction
type string "button" HTML button type: "button", "submit", "reset"

CSS Classes

Class Description
small Makes the button small
large Makes the button large
xlarge Makes the button extra large
outline Applies outline style
ghost Applies ghost style
rounded Applies rounded corners
pill Applies pill shape

Accessibility

Accessibility Features:
  • Keyboard navigation support (Tab, Enter, Space)
  • Screen reader friendly with semantic HTML
  • Focus states clearly visible
  • Disabled state properly communicated

Best Practices

  • Always use semantic <button> elements for actions
  • Provide clear, descriptive button text
  • Don't rely solely on color to convey button state
  • Ensure sufficient color contrast for text
  • Use appropriate button types in forms (type="submit" for form submission)
