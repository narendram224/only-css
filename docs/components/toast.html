Toast

Toasts are brief notifications that appear temporarily to provide feedback to users. Only-CSS provides toast components with multiple variants, positions, and auto-dismiss functionality.

Good to know: Toasts use CSS animations for smooth entrance and exit effects, providing elegant user feedback without JavaScript.

Basic Usage

The simplest way to create a toast is to use the oc-toast class:

Basic Toast
Success! Your changes have been saved.

Accessibility

Accessibility Features:
  • Semantic HTML structure
  • Proper color contrast
  • Screen reader friendly
  • Clear visual hierarchy

Best Practices

  • Keep toast messages brief and clear
  • Use appropriate variants for different message types
  • Position toasts consistently
  • Consider auto-dismiss timing
  • Test with screen readers

Design Guidelines

Pro Tip: Use toasts for non-critical feedback. For important information that requires user action, consider using dialogs or alerts instead.

When to Use Toasts

  • Success Feedback: Confirm successful actions
  • Error Messages: Show non-critical errors
  • Warnings: Alert to potential issues
  • Information: Provide helpful tips

Content Guidelines

  • Keep messages under 2 lines when possible
  • Use clear, action-oriented language
  • Include relevant icons for quick recognition
  • Test message clarity with users
