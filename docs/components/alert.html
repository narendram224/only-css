Alert

Alerts are used to communicate important messages to users, including success, warning, danger, and informational messages. Only-CSS provides semantic alert components with proper visual hierarchy and accessibility.

Good to know: Alerts use semantic HTML with proper ARIA attributes for screen readers and keyboard navigation.

API Reference

Attributes

Attribute Type Default Description
data-variant string "default" Alert variant: "default", "danger", "success", "warning"

CSS Classes

Class Description
oc-alert Base alert component class
oc-alert-title Alert title element (optional)
oc-alert-description Alert description element (optional)

Accessibility

Accessibility Features:
  • Semantic HTML structure for screen readers
  • Proper color contrast for all variants
  • Clear visual hierarchy with title and description
  • Keyboard accessible content

Best Practices

  • Use appropriate variants for different message types
  • Keep alert messages concise and actionable
  • Provide clear titles for structured alerts
  • Use success alerts for completed actions
  • Use danger alerts for errors or destructive actions
  • Use warning alerts for cautionary messages
  • Position alerts prominently but don't overuse them

Design Guidelines

Pro Tip: Use alerts sparingly to maintain their impact. Too many alerts can lead to alert fatigue where users ignore important messages.

When to Use Alerts

  • Success: After completing important actions (save, submit, delete)
  • Danger: For errors, validation failures, or critical issues
  • Warning: For cautionary messages or potential problems
  • Default: For general information or neutral messages

When Not to Use Alerts

  • For routine status updates
  • For non-critical information
  • For content that should always be visible
  • For complex interactive elements
