Badge

Badges are small components used to label, categorize, or organize items. Only-CSS provides a comprehensive badge system with multiple sizes, colors, styles, and positioning options.

Good to know: Badges use semantic HTML with proper ARIA attributes for screen readers and maintain consistent visual hierarchy across your application.

Accessibility

Accessibility Features:
  • Semantic HTML structure
  • Proper color contrast for all variants
  • Screen reader friendly
  • Keyboard accessible content

Best Practices

  • Use appropriate colors for semantic meaning
  • Keep badge text short and concise
  • Ensure sufficient color contrast
  • Use outline badges for less emphasis
  • Use counter badges for notifications only
  • Position badges consistently across your application

Design Guidelines

Pro Tip: Use color consistently across your application. Success badges should always indicate positive states, danger badges for negative states, etc.

When to Use Badges

  • Status Indicators: Show item status (active, inactive, pending)
  • Categories: Tag and categorize content
  • Notifications: Display unread counts or alerts
  • Labels: Provide additional context
  • Filters: Show active filters or tags

Content Guidelines

  • Keep text under 10 characters when possible
  • Use consistent terminology
  • Avoid abbreviations unless widely understood
  • Use numbers for counts and statistics
  • Consider localization for international users
