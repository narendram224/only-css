Avatar

Avatars are used to represent users or entities in your interface. Only-CSS provides a comprehensive avatar system with multiple sizes, shapes, status indicators, and group layouts.

Good to know: Avatars automatically handle fallback initials when no image is provided, ensuring consistent appearance across your application.

Accessibility

Accessibility Features:
  • Proper alt text for image avatars
  • Semantic HTML structure
  • Status indicators with proper color contrast
  • Keyboard accessible content

Best Practices

  • Always provide meaningful alt text for image avatars
  • Use consistent sizing within the same context
  • Ensure status indicators have sufficient color contrast
  • Use initials that represent the user's name
  • Consider loading states for image avatars
  • Maintain consistent avatar shapes across your application

Design Guidelines

Pro Tip: Use consistent avatar sizes throughout your application. Medium (2.5rem) is ideal for most use cases, while large (3rem) works well for user profiles.

When to Use Avatars

  • User Profiles: To represent individual users
  • Team Lists: To show team members or collaborators
  • Comments: To identify who made a comment
  • Notifications: To show who triggered an action
  • Chat Applications: To display conversation participants

Content Guidelines

  • Use 1-2 initials for text avatars
  • Use high-quality images for best results
  • Ensure images are square for consistent appearance
  • Use meaningful alt text for accessibility
  • Consider fallback colors for different users
