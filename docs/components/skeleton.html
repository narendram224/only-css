Skeleton

Skeleton screens are loading placeholders that show content structure while data is being loaded. Only-CSS provides skeleton components with animated shimmer effects for better perceived performance.

Good to know: Skeletons use pure CSS animations with shimmer effects, providing smooth loading feedback without JavaScript.

API Reference

CSS Classes

Class Description
oc-skeleton Base skeleton component class
circle Circular skeleton (for avatars, images)
title Title skeleton (larger, 60% width)
text Text skeleton (full width)
text-short Short text skeleton (70% width)

Accessibility

Accessibility Features:
  • Screen readers ignore skeleton elements
  • Maintains layout structure
  • Provides loading feedback
  • Reduces layout shift

Best Practices

  • Match skeleton layout to actual content
  • Use appropriate skeleton variants
  • Keep skeleton duration minimal
  • Test with assistive technologies
  • Consider loading state indicators

Design Guidelines

Pro Tip: Use skeletons for content that takes more than 1 second to load. They improve perceived performance and reduce layout shift.

When to Use Skeletons

  • Content Loading: Articles, posts, feeds
  • Data Fetching: User profiles, settings
  • Image Loading: Galleries, avatars
  • Form Loading: Dynamic forms, data entry
  • List Loading: Tables, task lists

Content Guidelines

  • Maintain consistent spacing
  • Use appropriate skeleton shapes
  • Consider content hierarchy
  • Test with real content
  • Monitor loading performance
