Themes

Only-CSS provides a flexible theming system built on CSS custom properties. Easily customize colors, typography, spacing, and more to match your brand or design system.

Good to know: Themes are built with CSS custom properties (variables) for maximum flexibility and performance. No preprocessor required!

Theme Overview

The theming system in Only-CSS is designed to be:

🎨 Customizable

Change colors, fonts, spacing, and more with simple CSS variables.

🌙 Dark Mode Ready

Built-in dark mode support that respects user preferences.

⚡ Performance

CSS variables provide excellent performance with no runtime overhead.

🔄 Consistent

Maintain design consistency across all components automatically.

Theme Customization

Creating a custom theme is as simple as overriding CSS variables:

Quick Custom Theme

/* Custom brand theme */ :root { --oc-color-primary: #8b5cf6; --oc-color-secondary: #ec4899; --oc-text-primary: #1f2937; --oc-bg-primary: #ffffff; --oc-border-color: #e5e7eb; --oc-font-sans: 'Inter', system-ui, sans-serif; --oc-border-radius: 0.5rem; } /* Apply theme to specific elements */ body.brand-theme { --oc-color-primary: #dc2626; --oc-color-secondary: #ea580c; }

Theme Switching

// JavaScript theme switching function setTheme(themeName) { document.body.className = themeName; } // Toggle between light and dark function toggleDarkMode() { document.body.classList.toggle('dark-theme'); } // Listen for system preference changes window.matchMedia('(prefers-color-scheme: dark)') .addEventListener('change', e => { if (e.matches) { document.body.classList.add('dark-theme'); } else { document.body.classList.remove('dark-theme'); } });

Component-Specific Theming

/* Theme specific components */ .ocean-theme .oc-button { --oc-button-primary-bg: #0ea5e9; --oc-button-primary-hover: #0284c7; } .forest-theme .oc-card { --oc-card-bg: #ecfdf5; --oc-card-border: #6ee7b7; } /* Custom component styles */ .custom-theme .oc-alert { --oc-alert-bg: #fef3c7; --oc-alert-border: #f59e0b; --oc-alert-text: #92400e; }

Dark Mode

Only-CSS includes built-in dark mode support that automatically respects user preferences:

Automatic Dark Mode: The library automatically detects user's color scheme preference and applies the appropriate theme.

Implementing Dark Mode

/* Dark mode using media query */ @media (prefers-color-scheme: dark) { :root { --oc-text-primary: #f9fafb; --oc-text-secondary: #d1d5db; --oc-bg-primary: #1f2937; --oc-bg-secondary: #111827; --oc-border-color: #374151; --oc-color-primary: #60a5fa; --oc-color-secondary: #9ca3af; } } /* Manual dark mode toggle */ .dark-theme { --oc-text-primary: #f9fafb; --oc-text-secondary: #d1d5db; --oc-bg-primary: #1f2937; --oc-bg-secondary: #111827; --oc-border-color: #374151; }

Dark Mode Best Practices

✅ Do

  • Test contrast ratios in dark mode
  • Adjust colors for reduced eye strain
  • Use darker backgrounds, not pure black
  • Consider reducing brightness of vibrant colors
  • Test on actual devices in low light

❌ Don't

  • Use pure black (#000000) backgrounds
  • Keep the same colors from light mode
  • Ignore contrast accessibility
  • Forget to test with content
  • Assume dark mode works without testing

Theme Toggle Component

Theme Toggle Example
function toggleTheme() { const body = document.body; const isDark = body.classList.contains('dark-theme'); if (isDark) { body.classList.remove('dark-theme'); localStorage.setItem('theme', 'light'); } else { body.classList.add('dark-theme'); localStorage.setItem('theme', 'dark'); } } // Load saved theme preference const savedTheme = localStorage.getItem('theme'); if (savedTheme === 'dark') { document.body.classList.add('dark-theme'); }
